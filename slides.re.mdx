# React Native とは

<Row>
  <Column align="left">
    <Text>Reactを使ってネイティブアプリを開発できる</Text>
    <Text>JSで iOS/Android 両方のアプリを作成</Text>
    <Text>WebView とは違い、直接 Native レイヤーを触りに行く</Text>
    <Text>開発元のMetaをはじめ、MS、Amazon、Shopify、Discord等大企業が導入</Text>
  </Column>
</Row>

# WebView
<Row>

# New Architecture

<Row>
  <Column align="left">
    <Text>ネイティブ層との通信を同期的に行う仕組み</Text>
    <Text>以前はJSON経由で全て非同期に行われていた</Text>
    <Text>2022 年から段階的に導入</Text>
    <Text>2024年でデフォルトで使われるように</Text>
    <Text>今年リリースされた0.82からLegacy Architectureの撤廃</Text>
  </Column>
</Row>

# Hermes V1

<Row>
  <Column align="left">
    <Text>JSC等に代わるJSEngine</Text>
    <Text>今年のReact Confでv1が発表された</Text>
    <Text>現状experimentalで使用可能</Text>
    <Text>es6のsyntaxがそのまま実行される</Text>
      <Text>以前のhermesではbabelでtranspileが必要だった</Text>
    <Text>V1 は現状 static hermes 非対応</Text>
      <Text>今後対応予定</Text>
      <Text>binary への compile等はその時にまた可能になる</Text>
    <Text>全体的には以前のhermesよりも速度が上がっている</Text>
  </Column>
</Row>

# Expo

<Row>
  <Column align="left">
    <Text>React NativeにおけるNext.js的な立ち位置</Text>
    <Text>ネイティブ層のビルドに必要なことを一通り担ってくれる</Text>
    <Text>以前は、Bluetoothなどexpoだと使えないネイティブの機能があった</Text>
    <Text>現在ではどんなサードパーティライブラリでも(基本的に)使用可能</Text>
      <Text>一部プラグインを自作する必要はある</Text>
  </Column>
</Row>

# 比較まとめ

<Row>
  <Column align="left">
    <Text>Expo router によってNext.jsのようなfile based routingに対応</Text>
    <Text>Tailwind等もWeb開発と似たような感覚で使える</Text>
    <Text>React Compilerも最新のExpoで自動で組み込まれている</Text>
    <Text>全体的に以前よりも格段に快適になった</Text>
    <Text>古いバージョンを使っている場合は最新バージョンへのアップデートは若干大変かも</Text>
  </Column>
</Row>

# AI開発との相性

<Row>
  <Column align="left">
    <Text>WebのReactと同じ書き方なので、基本的に相性は良い</Text>
    <Text>古いライブラリやバージョンとかを結構使われたりするので、initとかは公式が提供しているコマンドから始めるのが無難</Text>
    <Text>ライブラリのアップデートとかには不向きなのでこの辺は地道な作業が必要</Text>
  </Column>
</Row>
